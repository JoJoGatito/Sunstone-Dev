Fix RLS Policy Issue

Your events table UPDATE policy is missing the WITH CHECK clause, causing updates to silently fail.

This is why changes made in the admin interface appear to save but don't actually update in the database.

Test RLS Policies Fix RLS Policy

Manual Fix Instructions

If the automated fix tools aren't working, you can run this SQL in your Supabase SQL Editor:

-- Fix Events RLS Policy
DROP POLICY IF EXISTS "Allow update for authenticated users" ON public.events;

CREATE POLICY "Allow update for authenticated users"
ON public.events
FOR UPDATE
USING (auth.role() = 'authenticated')
WITH CHECK (auth.role() = 'authenticated');
